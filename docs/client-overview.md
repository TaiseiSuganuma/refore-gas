## 進行中の案件

### 1. registry-ocr（運用中）

**課題**: 不動産登記簿 PDF からの情報転記を事務担当が手作業で行っており負荷が大きい。

**ソリューション**: スプレッドシートのカスタムメニューから PDF をアップロード → GAS の OCR で「所在・地番・地目・地積・所有者氏名」を抽出 → シートへ自動追記。

詳細: [`projects/registry-ocr/docs/specification.md`](../projects/registry-ocr/docs/specification.md)

### 2. document-generator（Phase 0 完了、Phase 1 進行中 = 最初の主要案件）

**課題**: 仲介の **案件確定時に作成する書類が多く**、毎回事務担当が手作業で書類を作成・編集しており負荷が大きい（このプロジェクトを依頼された直接の動機）。

**スプレッドシート**: [【リフォレ】書類自動生成](https://docs.google.com/spreadsheets/d/12BEUxTqnADqWsmffGWos0tFe08FohEE-0RQJbLNAnEk/edit)

**書類リスト管理**: [リフォレ開発管理シート](https://docs.google.com/spreadsheets/d/1Am4A91sC05YPhq5gi2aBLe7zzjuPEp9pNy0xLh6hs8w/edit)

**North Star（目標形）** — 2026-05-27 設計変更:

「Google フォーム入力 → 案件マスタ（集約型・1案件=1行）に追記 → 行ヘッダのチェックボックスで案件複数選択 + 書類列チェックボックスで出力対象選択 → メニュー『書類発行』で一括 PDF 出力 → 案件ごとにフォルダを作って格納」

- 新規案件は **Google フォーム** から入力。修正は案件マスタ上で直接編集
- **1案件 = 1行 + 1フォルダ**（旧設計の「1案件=1スプレッドシート」は廃止）
- **案件マスタが Source of Truth**（冗長転記レイヤなし）
- 書類選択は **書類ごとに列を立ててチェックボックス**（パターン選択で初期チェック投入 + 個別調整可能）
- 出力対象は **行ヘッダのチェックボックス** で複数行選択。月末に「契約日=今月・ステータス=未出力」を絞って一括処理
- **ステータス**: `未着手 / 出力済み / 完了 / 保留`（プルダウン）
- **OCR（registry-ocr）はフォームに統合**（登記簿PDF添付欄から自動入力）
- 書類は **14種類**（A系統3 + B系統2 + 見積/請求/社内4 + 法務局5）

**月次集計の重要性**:
月末に書類が集中する運用。**案件マスタ 1 シートに全案件・全情報を集約**しておくことで、
QUERY や SUMIFS で「契約日=今月」「担当営業=X」などの集計が直接成立する。
旧 North Star（案件サマリーシート → マスタ自動転記）と比べて、データレイヤが 1 段減りシンプルになる。

**段階的アプローチ**:
- Phase 1: 既存「案件一覧」から **特定 1 案件・1 書類（土地売買契約書）** を PDF 出力する MVP
- Phase 2: 案件マスタの列拡張（パターン列・書類選択チェックボックス列・行頭選択チェックボックス） + 複数案件×複数書類の一括出力
- Phase 3: 明細繰返し対応、A 系統 B 系統の契約書実装
- Phase 4: **Google フォーム連携**（フォーム送信 → 案件マスタ自動追記 + 案件フォルダ自動作成）
- Phase 5: **OCR 統合**（フォームに登記簿 PDF 添付欄）
- Phase 6: **月次チェックリスト自動生成** + 見積書・請求書・法務局関連・社内シート対応

詳細: [`projects/document-generator/docs/specification.md`](../projects/document-generator/docs/specification.md)
