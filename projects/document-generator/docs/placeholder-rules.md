# プレースホルダ規約

Google Docs テンプレート内に書く差し込み記法のルール。
事務担当がテンプレートを編集する際もこのルールに従う。

> ⚠️ Phase 0 時点では暫定案。実物のテンプレートを見て確定する。

---

## 基本方針

- 記法は **Mustache / Handlebars 風** に統一
- テンプレ作成者（事務担当）が **覚えやすく、間違えにくい** ことを最優先
- Phase 1 では値差し込みのみを実装、条件分岐と繰返しは Phase 3 で実装

---

## 1. 値差し込み（Phase 1 で実装）

| 記法 | 例 | 説明 |
|---|---|---|
| `{{key}}` | `{{customer_name}}` | マスタシートの該当カラムの値で置換 |

### キー命名規則（暫定）

- 半角英小文字 + アンダースコア (`snake_case`)
- マスタシートの列名と 1:1 で対応させる
- 例: `customer_name`, `contract_date`, `property_location`

### 値の整形（フォーマット指定）

| 記法 | 例 | 出力例 |
|---|---|---|
| `{{key}}` | `{{contract_date}}` | `2026-05-17` （生の値） |
| `{{key:date_jp}}` | `{{contract_date:date_jp}}` | `令和8年5月17日` |
| `{{key:date}}` | `{{contract_date:date}}` | `2026年5月17日` |
| `{{key:yen}}` | `{{price:yen}}` | `¥1,200,000` |

> 実装時にどのフォーマットが必要かは、テンプレ実物を見て確定する。

### 未定義キーの扱い

- マスタに対応する列がない、または値が空の場合 → **TBD**
  - 案A: 空文字で置換し、警告ログを出す
  - 案B: エラーにして処理中断
  - 案C: プレースホルダのまま残し、人間が確認

## 2. 条件分岐（Phase 3 で実装）

```
{{#if is_corporate}}
法人名: {{company_name}}
{{/if}}
```

- 真偽値: マスタの値が「`真` / `TRUE` / `1` / `はい`」なら真、それ以外は偽
- ネスト・else 句のサポートは要検討（TBD）

## 3. 繰返し（Phase 3 で実装済み）

```
{{#each items}}
{{物件No}}.  {{所在}} {{地番}}
{{/each}}
```

実装ルール（templateService.expandRepeatBlocks_）:
- 開始マーカー `{{#each items}}` と終了マーカー `{{/each}}` は **単独の段落** として書く（同段落内の他テキスト混在は非対応）
- マーカー間の段落をテンプレートとして、`PlaceholderContext.items` 配列の各要素分だけ複製展開する
- 各複製でブロック内の `{{key}}` を `items[i][key]` で置換
- ネストはサポートしない
- 装飾（太字・下線等）は引き継がない（Phase 3 ではプレーンテキスト前提）
- `items` 未指定 or 空配列の場合はマーカーと本体を **削除** する（条件出力としても機能）

データソース:
- 法務局申請書の「不動産の表示」: 物件シートの 1 案件分（不動産番号 / 所在 / 地番 / 地目 / 地積）
- お客様契約書の「別紙詳細」: 物件シートの 1 案件分（物件No / 所在 / 地番）

詳細は [`documents/legal_documents_placeholders.md`](./documents/legal_documents_placeholders.md) および
[`documents/customer_contract_land_placeholders.md`](./documents/customer_contract_land_placeholders.md) 参照。

---

## 4. テンプレ作成時の注意

- Google Docs の **オートコレクト** が `{{` を全角に変える場合があるため、
  プレースホルダは確認してから保存する
- フォント・改行の都合でプレースホルダが分断されないよう、
  `{{customer_name}}` を **同じ書式設定で一気に入力** する
  （Docs では文字単位で書式が異なると `replaceText` でマッチしない場合がある）
