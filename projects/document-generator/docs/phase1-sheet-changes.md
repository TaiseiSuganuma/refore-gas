# Phase 1 着手時のシート編集内容

Phase 1（土地売買契約書 MVP）を成立させるため、既存スプレッドシート [【リフォレ】書類自動生成](https://docs.google.com/spreadsheets/d/12BEUxTqnADqWsmffGWos0tFe08FohEE-0RQJbLNAnEk/edit) に対して行うシート編集の指示書。

## 編集の流れ

ユーザー側で以下を手作業で実施:
1. 「案件一覧」シートに列を追加
2. 「明細」シートを「物件」シートに名称変更 + 列構成を見直し
3. 「設定」シートに必要な項目を追加

各セクションに **そのまま貼り付けられる見出し行** を載せたので、コピペで進められます。

---

## 1. 「案件一覧」シートに列追加

現在の列（A〜R）に続けて、後ろに 6 列追加。

### 追加する列（S〜X）

| 列 | カラム名 | 入力例 | 説明 |
|---|---|---|---|
| S | `契約日` | `2026/06/15` | 売買契約締結日（yyyy/MM/dd 形式） |
| T | `売主名義人` | （空 or `佐藤花子`） | 名義人が本人以外の場合のみ入力 |
| U | `売主続柄` | （空 or `配偶者`） | 名義人との続柄 |
| V | `手付金` | `50000` | 手付金額（半角数字、円） |
| W | `口座名義カナ` | `サトウテルオ` | 振込先名義のフリガナ |
| X | `支払予定日` | `2026/07/15` | 代金支払予定日 |

### 1行目のヘッダー（コピペ用）

S1 セル以降に貼り付け（タブ区切り）:

```
契約日	売主名義人	売主続柄	手付金	口座名義カナ	支払予定日
```

---

## 2. 「明細」シート → 「物件」シートへ再設計

### 名称変更

シートタブを **「明細」→「物件」** にリネーム。

### 新しい列構成

| 列 | カラム名 | 旧との対応 | 説明 |
|---|---|---|---|
| A | `案件ID` | 同名 | 案件一覧の `案件ID` と結合 |
| B | `物件No` | 旧「行No」 | 案件内の連番（1, 2, 3...） |
| C | `所在` | **新規分離** | 市区町村 + 大字（例: `南さつま市加世田内山田`） |
| D | `地番` | **新規分離** | 地番（例: `10647-1`） |
| E | `地目` | **新規分離** | 山林 / 田 / 畑 / 宅地 / 雑種地 等 |
| F | `面積(㎡)` | 同名 | 数値 |
| G | `雑抜き(㎡)` | 同名 | 数値 |
| H | `樹種` | 同名 | 杉 / 松 等 |
| I | `樹高(m)` | 同名 | レンジ表記可（`20〜25m`） |
| J | `金額(円)` | 同名 | 当該物件の金額 |
| K | `備考` | **新規** | 自由記述 |

### 1行目のヘッダー（コピペ用）

A1 セル以降に貼り付け:

```
案件ID	物件No	所在	地番	地目	面積(㎡)	雑抜き(㎡)	樹種	樹高(m)	金額(円)	備考
```

### 既存データの移行

サンプル `EST-20260309-001` の `南さつま市加世田内山田10647-1（立木）` を例にすると:

| 旧「住所（地目）」 | → 新「所在」 | 新「地番」 | 新「地目」 |
|---|---|---|---|
| 南さつま市加世田内山田10647-1（立木） | 南さつま市加世田内山田 | 10647-1 | 山林 (※「（立木）」は契約区分なので地目には入れない) |
| 〃（立木） | 南さつま市加世田内山田 | 10647-1 | 山林 |
| 南さつま市加世田内山田10510-1（立木） | 南さつま市加世田内山田 | 10510-1 | 山林 |

> **重要**: 旧「`〃`」を使った省略は廃止。すべての行に所在を明記する。
> 「（立木）」は契約区分（立木 / 土地 / 立木+土地）であり地目ではないため、地目列には登記情報の地目（山林等）を入れる。

| 項目 | 値 | 説明 |
|---|---|---|

---

## 4. 編集後のチェックポイント

- [ ] 「案件一覧」のヘッダ行が想定通り
- [ ] 「物件」シートに名称変更、列構成が新規構造
- [ ] サンプル案件 `EST-20260309-001` のデータが新形式で残っている（テストデータ）
- [ ] 「設定」シートに必要項目が揃っている

編集後、ユーザーから完了報告をもらってから Phase 1 の実装に進む。

---

## 5. 今回シートに追加しないもの（Phase 1 の判断）

- 「案件パターン」列（A①〜B②） → Phase 2 で追加
- 「契約状況」「売上(立木)」「売上(土地)」「利益」等の集計用列 → Phase 4 で追加（案件マスタ化と同時）
- 「契約日（和暦）」のような表示用列 → GAS 側で変換するため不要

> Phase 1 では最小限の変更に留め、Phase 4 で構造を本格再設計する。
