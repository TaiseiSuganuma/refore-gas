## 残りの作業（Phase 2 GAS 実装着手前に完了が必要）

### 作業 5: 「案件一覧」シートに取引先 ID 3 列を追加

#### 概要

取引先マスタ・代理人マスタを参照するため、案件一覧の末尾（AS〜AU）に ID 列 3 つを追加します。

#### 手順

AS 列以降に、以下のヘッダーを貼り付けてください（タブ区切り）:

```
売主取引先ID	買主取引先ID	代理人ID
```

#### 列ごとの設定（データ検証でプルダウン化）

| 列 | カラム名 | データ検証ソース | 備考 |
|---|---|---|---|
| AS | `売主取引先ID` | `取引先マスタ!A2:A` | 売主が法人の場合のみ入力。個人売主は空のまま（G〜K 列の手入力を使う） |
| AT | `買主取引先ID` | `取引先マスタ!A2:A` | B 系統で買主取引先を選択。A 系統では空（買主＝リフォレ固定） |
| AU | `代理人ID` | `代理人マスタ!A2:A` | 法務局申請書を作る案件で代理人を選択 |

設定方法: 列を選択 → データ → データの入力規則 → リストの範囲を指定。

#### サンプル案件 `EST-20260309-001` への入力

- A③（土地のみの仕入）なので **3 列とも空のまま** で OK。
- 法務局申請書も Phase 3 までは使わないので、`代理人ID` も空で問題なし。

### 作業 6: 「書類マスタ」シートに GAS 用 4 列を追加

#### 概要

GAS が書類を引き当てるための内部キーを追加します。
既存の 7 列（書類名/宛先/出力条件/原本/テンプレート/テンプレートURL/備考）はそのまま、その右に 4 列追加。

#### 手順

H 列以降に、以下のヘッダーを貼り付けてください:

```
書類ID	対応パターン	ファイル名テンプレ	有効
```

| 立木付土地売買契約書 | `purchase_contract_land_tree` | `A②` | `{{案件ID}}_立木付土地売買契約書_{{今日}}` | FALSE |

| 売上計算シート | `sales_calc` | `A①,A②,A③,B①,B②` | `{{案件ID}}_売上計算シート_{{今日}}` | FALSE |
| チェックリスト | `checklist` | `A①,A②,A③,B①,B②` | `{{案件ID}}_チェックリスト_{{今日}}` | FALSE |
| 法務局所有権移転＿登記原因証明情報 | `legal_transfer_reason` | `A②,A³,B²` | `{{案件ID}}_登記原因証明情報_{{今日}}` | FALSE |
| 法務局所有権移転＿委任状 | `legal_transfer_proxy` | `A②,A³,B²` | `{{案件ID}}_所有権移転委任状_{{今日}}` | FALSE |

> **「有効」列の運用**: Phase 2 の動作確認では土地売買契約書だけ TRUE にし、他は FALSE のまま。Phase 3 着手時に契約書系を順次 TRUE に切り替える。
>
> **K 列の設定**: 列を選択 → 挿入 → チェックボックス。

#### A 列「書類名」を案件一覧の書類列名と一致させる

案件一覧の書類列名（`書類_土地売買契約書` 等）と書類マスタの書類名を GAS が突合します。
既存の書類マスタ A 列の値はすでに案件一覧の書類列の suffix と一致しているはずなので、変更不要。

ただし、案件一覧 AJ〜AN の `書類_法務局_X1〜X5` は仮の列名のため、Phase 3 着手前に **実書類名に合わせて列名をリネーム** することを検討（例: `書類_法務局_X1` → `書類_法務局_登記原因証明情報`）。Phase 2 ではそのまま残置で OK。

### 作業 7: 「設定」シートにシート名 4 項目を追加

現在の設定シートに以下の 4 行を追加します。既存の 3 列構造（`項目 / 値 / 説明`）に従って入力。

| 項目 | 値 | 説明 |
|---|---|---|
| `書類マスタシート名` | `書類マスタ` | GAS が書類定義を引く際の参照先 |
| `取引先マスタシート名` | `取引先マスタ` | 伐採業者・買主マスタの参照先 |
| `代理人マスタシート名` | `代理人マスタ` | 法務局申請書の代理人マスタの参照先 |
| `選択列名` | `選択` | バッチ抽出に使う行頭チェック列（案件一覧 A 列）の名前 |

> **退役予定の項目**: `Phase1テンプレID_土地売買契約書` は Phase 2 以降、書類マスタの `テンプレートURL` から ID をパースして使うため使わなくなります。Phase 1 動作確認が終わったら削除可能（互換のため Phase 2 中は残置でも OK）。
