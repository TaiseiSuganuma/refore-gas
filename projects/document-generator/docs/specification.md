# document-generator 仕様書

スプレッドシートの案件情報をもとに、Google Docs / Sheets テンプレートへ差し込んで
PDF を指定 Drive フォルダに出力するシステム。

- **対象 GAS 種別**: Sheets コンテナバインド型
- **エントリースクリプト**: `src/Code.ts`（実装予定）
- **タイムゾーン**: `Asia/Tokyo`
- **GAS ランタイム**: V8
- **対象スプレッドシート**: [【リフォレ】書類自動生成](https://docs.google.com/spreadsheets/d/12BEUxTqnADqWsmffGWos0tFe08FohEE-0RQJbLNAnEk/edit) (fileId: `12BEUxTqnADqWsmffGWos0tFe08FohEE-0RQJbLNAnEk`)

---

## 1. 業務フローの理解（重要）

リフォレは **「山林を売りたい地主」と「買いたい企業や個人（または伐採業者）」の仲介** を行う会社。
売買は 2 段階で発生する。

```
[地主] ─(A: 仕入)─→ [リフォレ] ─(B: 売上)─→ [買主/伐採業者]
```

契約書は **A 系統（リフォレが甲＝買主）** と **B 系統（リフォレが甲＝売主）** の 2 系統。
さらに「土地が絡むか / 立木のみか」で分岐する。

### A 系統（仕入: 売主 → リフォレ）

| パターン | 対象 | 契約書 |
|---|---|---|
| A① | 立木のみ | 立木売買契約書 |
| A② | 立木 + 土地 | 立木付土地売買契約書 |
| A③ | 土地のみ | 土地売買契約書 |

### B 系統（売上: リフォレ → 買主）

| パターン | 対象 | 契約書 |
|---|---|---|
| B① | 立木のみ | お客様契約書_立木 |
| B② | 立木 + 土地 | お客様契約書_土地込（A③ 土地のみケースも含む） |

### 土地が絡むケースの追加書類（法務局関連 5 種類）

- 法務局所有権移転＿登記原因証明情報
- 法務局所有権移転＿委任状
- 法務局登記住所変更＿委任状
- 法務局所有権移転手続き＿登記申請書
- 法務局登記住所変更＿登記申請書

### 見積・請求・社内用

- お客様への御見積書（契約前、リフォレ→売主）
- お客様への請求書（リフォレ→買主、伐採業者から要求された場合）
- 売上計算シート（社内）
- チェックリスト（社内）

## 2. ユーザー操作フロー（予定）

```
[スプレッドシートを開く]
        ↓ onOpen()
[メニュー「書類出力」が表示される]
        ↓ クリック
[モーダルダイアログ起動]
   - アクティブ行の案件を確認 / 別案件を選ぶ
   - 案件のパターン（A①/A②/A③/B①/B②）を選択
     → 自動的に必要な書類セットがチェックされる
   - 必要に応じて書類を個別追加・除外
        ↓ 「PDF を生成」クリック
[各書類について]
   1. テンプレ Docs を複製
   2. プレースホルダを案件情報で置換（明細がある書類は表に行追加）
   3. PDF として案件ごとのフォルダに保存
   4. 一時 Docs を削除
        ↓
[ダイアログに結果表示（生成された PDF のリンク）]
```

### テンプレ形式の方針

- **Word(.docx) のテンプレは Google Docs に変換して移行**（事務担当の編集容易性と GAS 実装シンプル化のため）
- 移行作業はユーザー側で実施（または Drive 上で「Google Docs として開く」→保存）
- Sheets テンプレ（見積書・請求書等）は Sheets のまま扱い、PDF 化する

## 8. エラーハンドリング方針

- マスタシートに必須値が無い場合は **生成を中断し、不足列を表示**
- テンプレに存在するプレースホルダがマスタに無い場合は **警告ログ**（処理は継続）
- 一時 Docs はエラー時も `finally` で削除

---

## 9. ログ

- `Logger_` 名前空間経由で Stackdriver に転送
- 各書類の生成開始・終了、エラー、生成 PDF の Drive URL を記録

## 11. 既存実装との関係

案件一覧シートの「使い方」には `generateEstimatePdfFromActiveRow` という関数名と
「ボタンに割り当てる」記述があり、見積書 PDF 出力の **既存実装または設計** が存在する。

**方針（確定済み）**: このプロジェクトで **既存の見積書実装も含めて作り直す**。
document-generator に一本化し、既存 GAS は段階的に置き換える。
