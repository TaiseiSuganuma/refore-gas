# document-generator 仕様書

スプレッドシートの案件情報をもとに、Google Docs テンプレートへ差し込んで
PDF を指定 Drive フォルダに出力するシステム。

- **対象 GAS 種別**: Sheets コンテナバインド型
- **エントリースクリプト**: `src/Code.ts`（実装予定）
- **タイムゾーン**: `Asia/Tokyo`
- **GAS ランタイム**: V8

> ⚠️ このドキュメントは **Phase 0（情報収集・仕様化）** の初期スケルトンです。
> マスタシートとテンプレートを確認しながら順次埋めていきます。
> 「TBD」と書かれた箇所は確定後に更新します。

---

## 1. ユーザー操作フロー（予定）

```
[スプレッドシートを開く]
        ↓ onOpen()
[メニュー「書類出力」が表示される]
        ↓ クリック
[モーダルダイアログ起動]
   - 案件を選択（マスタシートから）
   - 出力したい書類をチェックボックスで複数選択（最大6件）
        ↓ 「PDF を生成」クリック
[各書類について]
   1. テンプレ Docs を複製
   2. プレースホルダを案件情報で置換
   3. PDF として指定 Drive フォルダに保存
   4. 一時 Docs を削除
        ↓
[ダイアログに結果を表示（生成された PDF へのリンク含む）]
```

## 6. 使用 OAuth スコープ（暫定）

| スコープ | 用途 |
|---|---|
| `spreadsheets` | マスタシートの読み取り |
| `drive` | テンプレ複製・PDF 化・指定フォルダへの保存 |
| `documents` | Google Docs のプレースホルダ置換 |
| `script.container.ui` | メニュー・ダイアログ表示 |

スコープ追加が必要な場合は共通 [`CLAUDE.md`](../../../CLAUDE.md) のルールに従い事前確認する。

---

## 7. エラーハンドリング方針

- マスタシートに必要な値が無い場合は **生成を中断し、どの列が不足かを表示する**
- テンプレに存在するプレースホルダがマスタに無い場合は **警告**（処理は継続、未差込のまま PDF を出すか中断するかは Phase 2 で決定）
- 一時 Docs はエラー時も削除する（`finally` で `setTrashed(true)`）

---

## 8. ログ

- `Logger_` 名前空間経由で Stackdriver に転送
- 各書類の生成開始・終了、エラー、生成された PDF の Drive URL を記録
