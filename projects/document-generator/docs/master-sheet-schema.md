# マスタシート / 案件サマリーシート スキーマ

書類生成の入力源となるスプレッドシートの構造。

Phase 1〜3 では既存の **集約型マスタシート** を使用し、
Phase 4 で **「1案件 = 1ファイル」の案件サマリーシート** 構造に再設計する。

- **Phase 1〜3 の対象**: [【リフォレ】書類自動生成](https://docs.google.com/spreadsheets/d/12BEUxTqnADqWsmffGWos0tFe08FohEE-0RQJbLNAnEk/edit) (fileId: `12BEUxTqnADqWsmffGWos0tFe08FohEE-0RQJbLNAnEk`)
- **Phase 4 以降の対象**: フォーム送信で自動生成される案件サマリーシート（テンプレからコピー）

---

## 1. シート構成（実物確認済み）

| シート名 | 役割 | 状態 |
|---|---|---|
| 案件一覧 | 1 案件 1 行で案件情報を保持 | **既存** |
| 明細 | 案件 ID で紐づく物件・樹種別の明細行（繰返し項目） | **既存** |
| 設定 | 出力先フォルダ ID、テンプレシート名等 | **既存** |
| 見積書 | 見積書のレイアウトシート（PDF 元データ） | **既存** |
| 書類マスタ | 書類ごとのテンプレ ID・出力条件・命名規則 | **新設予定** |

---

## 2. 「案件一覧」シート

1 案件 = 1 行。すべての書類で参照される基本情報。

| 列 | カラム名 | 型 | 説明 | 必須 |
|---|---|---|---|---|
| A | `案件ID` | 文字列 | 案件の一意 ID（例: `EST-20260309-001`） | ✓ |
| B | `見積日` | 日付 | `yyyy/MM/dd` | ✓ |
| C | `宛名` | 文字列 | 顧客名（例: 佐藤てるお） | ✓ |
| D | `敬称` | 文字列 | 様 / 御中 など | ✓ |
| E | `会社名` | 文字列 | 顧客の所属企業（個人なら空） | |
| F | `郵便番号` | 文字列 | ハイフン付き（例: `893-0025`） | |
| G | `住所` | 文字列 | 顧客住所 | |
| H | `TEL` | 文字列 | 電話番号 | |
| I | `担当者` | 文字列 | リフォレ側担当者（例: 梅下 生光） | |
| J | `備考` | 文字列 | 自由記述 | |
| K | `PDFファイル名` | 文字列 | 既存実装で使われている見積書 PDF のファイル名 | |
| L | `銀行名` | 文字列 | 売主の振込先銀行 | |
| M | `支店名` | 文字列 | 支店名 | |
| N | `種類` | 文字列 | 口座種別（普通 / 当座） | |
| O | `口座番号` | 文字列 | 口座番号 | |
| P | `口座名義` | 文字列 | 口座名義（カタカナ） | |
| Q | `見積金額(円)` | 数値/文字列 | 見積総額 | |
| R | `ステータス` | 文字列 | 案件のステータス（運用フローに従う） | |

### サンプル

```
EST-20260309-001, 2026/03/09, 佐藤てるお, 様, 株式会社リフォレ, 893-0025,
鹿児島県鹿屋市西祓川町509番地5, 080-8570-3872, 梅下 生光,
※運搬費用、伐採費用等は差し引いた金額になっております。,
御見積書_EST-20260309-002.pdf, テスト銀行, テスト支店, 普通, 1234567, テストタロウ, 500000, ...
```

## 3. 「明細」シート

案件 ID で紐づく繰返し項目。1 案件 = 複数行。

| 列 | カラム名 | 型 | 説明 |
|---|---|---|---|
| A | `案件ID` | 文字列 | 案件一覧の `案件ID` と結合 |
| B | `行No` | 数値 | 1 から連番 |
| C | `住所（地目）` | 文字列 | 物件の所在地と地目（例: `南さつま市加世田内山田10647-1（立木）`） |
| D | `樹種` | 文字列 | 杉 / 松 など |
| E | `樹高(m)` | 文字列 | レンジ表記もあり（例: `20〜25m`） |
| F | `面積(㎡)` | 数値 | 面積。空欄あり |
| G | `雑抜き(㎡)` | 数値 | 雑抜き面積 |
| H | `金額(円)` | 数値 | 当該行の金額 |

### サンプル

```
EST-20260309-001, 1, 南さつま市加世田内山田10647-1（立木）, 杉, 20〜25m, 7986, 2833, 155000
EST-20260309-001, 2, 〃（立木）, 松, 15〜20m, , 4918, 175000
...
```

「`〃`」は前行と同じ住所を意味する。表示時はそのまま使い、データ抽出時には注意。

## 5. 「書類マスタ」シート（新設予定）

書類ごとの設定を 1 行 1 書類で保持する。

| 列 | カラム名 | 型 | 説明 |
|---|---|---|---|
| A | `書類ID` | 文字列 | 内部識別子（例: `purchase_contract_land`） |
| B | `書類名` | 文字列 | UI 表示名（例: 土地売買契約書） |
| C | `系統` | 文字列 | `A①` / `A②` / `A③` / `B①` / `B②` / `共通` |
| D | `テンプレートID` | 文字列 | Google Docs / Sheets の fileId |
| E | `テンプレ種別` | 文字列 | `docs` / `sheets` |
| F | `出力フォルダID` | 文字列 | 書類別の出力先（空欄なら共通フォルダ） |
| G | `ファイル名テンプレ` | 文字列 | 命名規則（例: `{{案件ID}}_土地売買契約書_{{今日}}.pdf`） |
| H | `明細含む` | 真偽 | 明細シートを表として差し込むか |
| I | `備考` | 文字列 | |

> このシートは Phase 1 着手時にユーザーが手動で 1 行（土地売買契約書）から作成する。
> その後の書類追加時に同様に行を追加していく。

## 6. 確認・確定が必要な項目（Phase 1〜2）

- [ ] **案件のパターン判定方法**: A①〜B② をどう決めるか
  - 案A: 案件一覧シートに「パターン」列を追加（**Phase 2 で採用予定**）
  - 案B: 「土地フラグ」「立木フラグ」を追加して導出
  - 案C: ダイアログで毎回人間が選ぶ
- [ ] **ファイル名規則**: `{案件ID}_{書類名}_{日付}.pdf` で問題ないか
- [ ] **乙（リフォレ側）情報の管理**:
  - 現状はテンプレに固定で書かれている（住所・代表者名・TEL 等）
  - 設定シートに切り出して動的化するか？（担当者の変更があり得るため検討）
- [ ] **金額の和暦/カンマ表記**: 契約書では「金五十万円」のような書式が必要か

## 7. 案件サマリーシート構造（Phase 4 で再設計）

Phase 4 でフォーム連携を導入する際、**1案件 = 1スプレッドシートファイル** の構造に移行する。
テンプレからコピーされた状態で各案件フォルダに配置される。

### 想定するシート構成（テンプレ）

| シート名 | 役割 |
|---|---|
| 案件基本情報 | 案件ID、パターン、契約日、リフォレ側担当者、ステータス |
| 売主情報 | 売主の氏名・住所・連絡先・振込口座（A 系統の甲） |
| 買主情報 | 買主の氏名・住所・連絡先（B 系統の乙、伐採業者など） |
| 土地情報 | 所在・地番・地目・地積・筆数（複数物件対応、繰返し可） |
| 立木情報 | 物件ごとの樹種・樹高・面積・雑抜き・金額（明細相当） |
| 金額条件 | 売買金額、手付金、支払予定日、消費税の取り扱い |
| 法務局情報 | 法務局関連書類で必要な追加情報（土地絡みの場合のみ） |
| 生成書類リスト | この案件で生成された PDF のリンク履歴 |

### 全社の案件マスタ

別ファイルとして「**案件一覧マスタ**」スプレッドシートを Drive 共通フォルダに配置。
1案件 1行で以下を保持する想定:

| 列 | カラム名 | 説明 |
|---|---|---|
| A | `案件ID` | 一意 ID |
| B | `案件名` | 顧客名 + 物件名等 |
| C | `パターン` | A①〜B② |
| D | `担当者` | リフォレ側担当者 |
| E | `ステータス` | 見積中 / 契約済 / 完了 |
| F | `フォルダURL` | 案件フォルダへのリンク |
| G | `サマリーシートURL` | 案件サマリーシートへのリンク |
| H | `作成日時` | フォーム送信日時 |
| I | `更新日時` | 最終更新日時 |

> ↑ Phase 4 着手時にユーザーと最終確定。
> 現状の「案件一覧」シートからの移行手順も Phase 4 で設計する。
