# 案件マスタ / 関連シート スキーマ

書類生成の入力源となるスプレッドシートの構造。

**2026-05-27 改訂**: 旧「1案件=1スプレッドシート」方式から **「1案件=1行+1フォルダ」の集約型** に変更。
本書は新方針の最終形を記載。旧 § 7「案件サマリーシート」概念は廃止された（[`specification.md` § 12](./specification.md#12-設計変更履歴) 参照）。

- **対象スプレッドシート**: [【リフォレ】書類自動生成](https://docs.google.com/spreadsheets/d/12BEUxTqnADqWsmffGWos0tFe08FohEE-0RQJbLNAnEk/edit) (fileId: `12BEUxTqnADqWsmffGWos0tFe08FohEE-0RQJbLNAnEk`)
  - Phase 4 で Google フォーム連携時、別ファイルにマイグレーションする可能性あり
- 案件マスタ自体は **このスプレッドシート内の「案件一覧」シート → 「案件マスタ」シート** に拡張する形（Phase 2 で改名・列拡張）

---

## 1. シート構成

| シート名 | 役割 | 状態 |
|---|---|---|
| 案件一覧 → 案件マスタ | **1 案件 1 行で全情報を集約** | Phase 1 では「案件一覧」のまま、Phase 2 で「案件マスタ」に改名・列拡張 |
| 物件 | 案件 ID で紐づく物件・樹種別の明細行（繰返し項目） | 既存「明細」シートを Phase 1 で「物件」に再設計 |
| 設定 | 出力先フォルダ ID、テンプレシート名、書類マスタシート名等 | 既存 |
| 出力書類リスト | 14 書類の【原本】【テンプレ】Docs URL を保持。GAS の入力源ではなく **人間が運用する書類一覧** | 2026-05-27 移行済み |
| 書類マスタ | 書類ごとのテンプレ ID・対応パターン・命名規則（GAS の入力源） | **Phase 2 で新設**（出力書類リストから手動転記） |
| 取引先マスタ | 伐採業者・買主など法人の住所・代表者・口座情報 | **Phase 2 で新設**（[§ 9](#9-取引先マスタシートphase-2-新設) 参照） |
| 代理人マスタ | 法務局申請書の代理人欄に入る個人 | **Phase 2 で新設**（[§ 10](#10-代理人マスタシートphase-2-新設) 参照） |
| 見積書 | 見積書のレイアウトシート（PDF 元データ） | 既存。Phase 6 で document-generator に統合 |

> **売主（地主）情報の管理方針**: 売主は 1 案件限りのケースが多いため **案件マスタの列に直接記載**（マスタ化しない）。
> **自社（リフォレ）情報**: 担当者変更頻度が低いため **テンプレに固定文字列で残す**（マスタ化しない）。変更時はテンプレを人間が手動編集。

## 3. 「物件」シート（旧「明細」シート）

案件 ID で紐づく繰返し項目。1 案件 = 複数行。

### Phase 1 で再設計

理由: 旧「住所（地目）」列が `南さつま市加世田内山田10647-1（立木）` のような形式で
所在・地番・地目（本来）が混在していたため、契約書差し込みのために分離する。

詳細は [`phase1-sheet-changes.md`](./phase1-sheet-changes.md) 参照。

### 新しい列構成

| 列 | カラム名 | 型 | 説明 |
|---|---|---|---|
| A | `案件ID` | 文字列 | 案件マスタの `案件ID` と結合 |
| B | `物件No` | 数値 | 案件内の連番（1, 2, 3...） |
| C | `所在` | 文字列 | 市区町村+大字（例: `南さつま市加世田内山田`） |
| D | `地番` | 文字列 | 地番（例: `10647-1`） |
| E | `地目` | 文字列 | 山林 / 田 / 畑 / 宅地 / 雑種地 等 |
| F | `面積(㎡)` | 数値 | 面積 |
| G | `雑抜き(㎡)` | 数値 | 雑抜き面積 |
| H | `樹種` | 文字列 | 杉 / 松 等 |
| I | `樹高(m)` | 文字列 | レンジ表記可（例: `20〜25m`） |
| J | `金額(円)` | 数値 | 当該物件の金額 |
| K | `備考` | 文字列 | 自由記述 |

### サンプル

```
案件ID            物件No  所在                    地番      地目  面積(㎡)  雑抜き(㎡)  樹種  樹高(m)    金額(円)
EST-20260309-001  1       南さつま市加世田内山田  10647-1   山林  7986      2833        杉    20〜25m    155000
EST-20260309-001  2       南さつま市加世田内山田  10647-1   山林              4918        松    15〜20m    175000
EST-20260309-001  3       南さつま市加世田内山田  10510-1   山林  2571      1190        杉    20〜25m    65000
```

旧形式での `〃` 省略は廃止。すべての行に所在を明記する。
「（立木）」は案件パターンで管理するため物件シートには入れない。

## 9. 取引先マスタシート（Phase 2 新設）

伐採業者・買主など、**【原本】テンプレに「ほぼ固定で印字されていた」法人情報** を別シートで管理し、案件マスタからプルダウン参照する。

### 背景

紙運用時代の【原本】には、よく取引する伐採業者（例: 株式会社 陽）の社名・住所・代表者名・登録番号が
あらかじめ印字されていた。実運用では「ほぼこの会社に依頼」だが、案件によって差し替えるケースもある。
【テンプレ】化するときに固定値で残すと差し替えが手作業になるため、**マスタから引いてプレースホルダに展開する** 構造にする。

### 列構成

| 列 | カラム名 | 型 | 説明 |
|---|---|---|---|
| A | `取引先ID` | 文字列 | 内部識別子（例: `PARTNER-001`） |
| B | `区分` | プルダウン | `伐採業者` / `買主` / `その他法人` |
| C | `会社名` | 文字列 | 例: 株式会社 陽 |
| D | `代表者名` | 文字列 | 例: 加塩 忠勇 |
| E | `住所` | 文字列 | 例: 鹿児島県志布志市志布志町帖3946-8 |
| F | `連絡先` | 文字列 | 電話番号 |
| G | `登録番号` | 文字列 | インボイス登録番号（例: T6340001015461） |
| H | `銀行名` | 文字列 | 取引先側口座（必要なら） |
| I | `支店名` | 文字列 | |
| J | `口座種別` | 文字列 | 普通 / 当座 |
| K | `口座番号` | 文字列 | |
| L | `口座名義` | 文字列 | |
| M | `口座名義カナ` | 文字列 | |
| N | `備考` | 文字列 | |

### 案件マスタ側の参照列

案件マスタに以下の列を追加し、`取引先ID` のプルダウンを設置:

| カラム名 | 説明 |
|---|---|
| `買主取引先ID` | B 系統で買主側の取引先。A 系統では空（買主＝リフォレ固定） |
| `売主取引先ID` | A 系統で売主が法人の場合のみ。個人売主は案件マスタの売主列を使用 |

## 11. 自社（リフォレ）情報の扱い

各契約書テンプレの **乙欄**（A 系統）または **売主欄**（B 系統）に書かれているリフォレ情報は、
**テンプレに固定文字列で残す**（マスタ化しない）。

理由（2026-05-27 ユーザー判断）:
- 担当営業・代表者の変更頻度は低い
- 変更時はテンプレを人間が手動で編集すれば足りる
- マスタ化のコストに見合うメリットが少ない

固定で残す情報（参考）:
