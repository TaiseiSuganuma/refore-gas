# マスタシート スキーマ

書類生成の入力源となるスプレッドシートの構造を記述する。

> ⚠️ Phase 0 時点では未確定。実物を確認して埋める。

---

## 1. シート構成（暫定）

マスタシートは複数のシートタブで構成される想定。

| シート名 | 役割 | 状態 |
|---|---|---|
| 案件マスタ | 1 案件 1 行で案件情報を保持 | TBD |
| 書類マスタ | 出力可能な書類の一覧（テンプレID、出力条件、保存先など） | TBD |
| 物件マスタ | 案件と紐づく物件情報（必要な場合） | TBD |
| 関係者マスタ | 売主・買主・保証人など（繰返し項目用） | TBD |

## 2. 「案件マスタ」シート

1 案件 = 1 行。すべての書類のプレースホルダ値の元となるシート。

| 列 | カラム名（仮） | 型 | 説明 | 必須 |
|---|---|---|---|---|
| A | `案件ID` | 文字列 | 案件の一意 ID | ✓ |
| B | `案件名` | 文字列 | 内部管理用の名称 | |
| ... | TBD | | | |

> 実物のシートを共有してもらった後に確定する。

---

## 3. 「書類マスタ」シート（暫定設計）

出力対象書類の一覧と、その出力条件・テンプレ・保存先を管理するシート。

| 列 | カラム名 | 型 | 説明 |
|---|---|---|---|
| A | `書類ID` | 文字列 | 内部識別子（例: `sales_contract`） |
| B | `書類名` | 文字列 | UI に表示する名前（例: 売買契約書） |
| C | `テンプレートID` | 文字列 | Google Docs テンプレの fileId |
| D | `出力条件` | 文字列/式 | 出力可否の条件（例: `売買金額 > 0`） |
| E | `出力フォルダID` | 文字列 | 保存先 Drive フォルダの ID |
| F | `ファイル名テンプレ` | 文字列 | 命名規則（例: `{{案件ID}}_売買契約書_{{今日}}.pdf`） |
| G | `備考` | 文字列 | 自由記述 |

> ↑ 暫定案。実物のシート構成を見て調整する。

---

## 4. プレースホルダとマスタカラムの対応

各書類のプレースホルダがマスタのどのカラムを参照するかを `documents/<書類名>.md` に記載する。
